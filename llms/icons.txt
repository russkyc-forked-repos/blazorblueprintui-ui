# BlazorUI Icon Libraries

BlazorUI provides **three separate icon packages** to suit different design preferences and project needs. All packages offer beautiful, customizable SVG icons as Blazor components.

---

## Icon Library Comparison

| Package | Icons | Variants | Size | License | Best For |
|---------|-------|----------|------|---------|----------|
| **BlazorUI.Icons.Lucide** | 1,640 | 1 (stroke) | 24x24 | ISC | Modern, clean interfaces |
| **BlazorUI.Icons.Heroicons** | 1,288 | 4 (outline, solid, mini, micro) | 16-24px | MIT (Tailwind Labs) | Tailwind-based designs, flexible sizing |
| **BlazorUI.Icons.Feather** | 286 | 1 (stroke) | 24x24 | MIT | Simple, lightweight projects |

---

## Installation

Install one or more icon packages:

```bash
# Lucide Icons (1,640 stroke-based icons)
dotnet add package BlazorUI.Icons.Lucide

# Heroicons (1,288 icons with 4 variants)
dotnet add package BlazorUI.Icons.Heroicons

# Feather Icons (286 minimalist stroke-based icons)
dotnet add package BlazorUI.Icons.Feather
```

---

## Setup

Add the namespace for your chosen icon library to `_Imports.razor`:

```razor
@* For Lucide Icons *@
@using BlazorUI.Icons.Lucide.Components

@* For Heroicons *@
@using BlazorUI.Icons.Heroicons.Components

@* For Feather Icons *@
@using BlazorUI.Icons.Feather.Components
```

## 1. Lucide Icons

### Basic Usage

```razor
<LucideIcon Name="camera" />
<LucideIcon Name="heart" Size="32" Color="red" />
<LucideIcon Name="check" Class="text-green-500" />
```

### API Reference

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Name` | `string` | **Required** | Icon name in kebab-case (e.g., "camera", "chevron-down") |
| `Size` | `int` | `24` | Icon size in pixels |
| `Color` | `string` | `"currentColor"` | CSS color value |
| `StrokeWidth` | `double` | `2.0` | SVG stroke width |
| `Class` | `string` | `null` | Additional CSS classes |
| `AriaLabel` | `string` | `null` | Accessibility label |

### Examples

```razor
@* Basic icon *@
<LucideIcon Name="home" />

@* Customized icon *@
<LucideIcon
    Name="settings"
    Size="20"
    Color="#3B82F6"
    StrokeWidth="1.5"
    Class="rotate-90" />

@* In a button *@
<Button>
    <LucideIcon Name="plus" Size="16" />
    Add Item
</Button>

@* With loading state *@
@if (isLoading)
{
    <LucideIcon Name="loader-2" Size="16" Class="animate-spin" />
}
```

### Popular Lucide Icon Names

**Navigation:** `home`, `menu`, `x`, `chevron-down`, `chevron-up`, `arrow-left`, `arrow-right`
**Actions:** `plus`, `minus`, `edit`, `trash`, `save`, `download`, `upload`, `search`
**Status:** `check`, `check-circle`, `x-circle`, `alert-circle`, `alert-triangle`, `info`
**User:** `user`, `user-plus`, `users`, `log-in`, `log-out`, `lock`, `unlock`
**Media:** `image`, `file`, `folder`, `camera`, `video`, `music`

**Browse all:** https://lucide.dev/icons

## 2. Heroicons

### Basic Usage

```razor
@* Outline variant (default) *@
<HeroIcon Name="camera" />

@* Solid variant *@
<HeroIcon Name="camera" Variant="HeroIconVariant.Solid" />

@* Mini variant (20x20) *@
<HeroIcon Name="camera" Variant="HeroIconVariant.Mini" />

@* Micro variant (16x16) *@
<HeroIcon Name="camera" Variant="HeroIconVariant.Micro" />
```

### Icon Variants

Heroicons uniquely offers **4 variants** of each icon:

- **Outline** (24x24) - Stroke-based, 1.5px stroke *(default)*
- **Solid** (24x24) - Filled paths
- **Mini** (20x20) - Filled paths, smaller details
- **Micro** (16x16) - Filled paths, simplified

### API Reference

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Name` | `string` | **Required** | Icon name in kebab-case (e.g., "camera", "home") |
| `Variant` | `HeroIconVariant` | `Outline` | Icon variant (Outline, Solid, Mini, Micro) |
| `Size` | `int?` | `null` | Override default size (24, 24, 20, or 16 based on variant) |
| `Color` | `string` | `"currentColor"` | CSS color value |
| `StrokeWidth` | `double?` | `1.5` | SVG stroke width (outline only) |
| `Class` | `string` | `null` | Additional CSS classes |
| `AriaLabel` | `string` | `null` | Accessibility label |

### Examples

```razor
@* Outline icon (default) *@
<HeroIcon Name="home" />

@* Solid icon for emphasis *@
<HeroIcon Name="star" Variant="HeroIconVariant.Solid" Color="#FFD700" />

@* Mini icon for compact UI *@
<HeroIcon Name="check-circle" Variant="HeroIconVariant.Mini" Class="text-green-500" />

@* Micro icon for dense layouts *@
<HeroIcon Name="x-mark" Variant="HeroIconVariant.Micro" Size="14" />

@* In a button with solid variant *@
<Button>
    <HeroIcon Name="heart" Variant="HeroIconVariant.Solid" Size="16" />
    Favorite
</Button>

@* Status indicators using different variants *@
<Badge>
    <HeroIcon Name="check" Variant="HeroIconVariant.Micro" Size="14" />
    Verified
</Badge>
```

### Variant Usage Guidelines

```razor
@* Outline: General UI, navigation *@
<nav>
    <HeroIcon Name="home" Variant="HeroIconVariant.Outline" />
    <span>Dashboard</span>
</nav>

@* Solid: Active states, emphasis *@
<Button Variant="primary">
    <HeroIcon Name="bookmark" Variant="HeroIconVariant.Solid" />
    Saved
</Button>

@* Mini: Inline with text, badges *@
<Badge>
    <HeroIcon Name="bell" Variant="HeroIconVariant.Mini" />
    Notifications
</Badge>

@* Micro: Very small UI, dense layouts *@
<div class="text-xs">
    <HeroIcon Name="clock" Variant="HeroIconVariant.Micro" Size="12" />
    2m ago
</div>
```

### Popular Heroicons Names

**Actions:** `plus`, `minus`, `pencil`, `trash`, `archive`, `arrow-down-tray`, `arrow-up-tray`
**Status:** `check`, `check-circle`, `x-circle`, `exclamation-circle`, `exclamation-triangle`

## Common Use Cases

### Icons in Buttons

```razor
@using BlazorUI.Components

@* Lucide *@
<Button>
    <LucideIcon Name="plus" Size="16" />
    Add Item
</Button>

@* Heroicons (Solid for emphasis) *@
<Button Variant="default">
    <HeroIcon Name="plus" Variant="HeroIconVariant.Solid" Size="16" />
    Add Item
</Button>

@* Feather *@
<Button>
    <FeatherIcon Name="plus" Size="16" />
    Add Item
</Button>

@* Icon-only button *@
<Button Size="icon" Variant="ghost">
    <LucideIcon Name="more-horizontal" Size="20" />
</Button>
```

### Icons in Navigation

```razor
<nav class="space-y-2">
    @* Using Lucide *@
    <a href="/" class="flex items-center space-x-2">
        <LucideIcon Name="home" Size="20" />
        <span>Dashboard</span>
    </a>

    @* Using Heroicons with Mini variant for compact nav *@
    <a href="/projects" class="flex items-center space-x-2">
        <HeroIcon Name="folder" Variant="HeroIconVariant.Mini" />
        <span>Projects</span>
    </a>

    @* Using Feather *@
    <a href="/settings" class="flex items-center space-x-2">
        <FeatherIcon Name="settings" Size="20" />
        <span>Settings</span>
    </a>
</nav>
```

### Icons in Dropdown Menus

```razor
<DropdownMenu>
    <DropdownMenuTrigger AsChild>
        <Button Variant="outline">
            Actions
            <LucideIcon Name="chevron-down" Size="16" />
        </Button>
    </DropdownMenuTrigger>
    <DropdownMenuContent>
        <DropdownMenuItem>
            <LucideIcon Name="edit" Size="16" />
            Edit
        </DropdownMenuItem>
        <DropdownMenuItem>
            <HeroIcon Name="document-duplicate" Variant="HeroIconVariant.Mini" />

        <DropdownMenuItem>
            <FeatherIcon Name="trash-2" Size="16" />
            Delete
        </DropdownMenuItem>
    </DropdownMenuContent>

```

## Choosing an Icon Library

### Use **Lucide** when:
- You want a large selection (1,640 icons)
- You prefer consistent stroke-based design
- You're building modern, clean interfaces
- You need a familiar icon set (similar to Feather, but more extensive)

### Use **Heroicons** when:
- You're using Tailwind CSS (designed by Tailwind Labs)
- You need multiple variants (outline, solid, mini, micro)
- You want flexibility in icon sizing for different contexts
- You need both stroke and filled icon styles

### Use **Feather** when:
- You want a lightweight, minimal icon set
- 286 carefully curated icons is sufficient
- You prefer simple, minimalist stroke-based design
- You want a smaller bundle size

### Mix and Match

You can install multiple icon libraries and use them together:

```razor
@using BlazorUI.Icons.Lucide.Components
@using BlazorUI.Icons.Heroicons.Components
@using BlazorUI.Icons.Feather.Components

@* Use Heroicons for navigation (Tailwind-designed) *@
<nav>
    <HeroIcon Name="home" Variant="HeroIconVariant.Outline" />
</nav>

@* Use Lucide for UI actions (large selection) *@
<Button>
    <LucideIcon Name="sparkles" Size="16" />
    Generate
</Button>

@* Use Feather for simple indicators *@
<FeatherIcon Name="check-circle" Class="text-green-500" />
```

## Styling Icons

### Using Tailwind CSS Classes

```razor
@* Color *@
<LucideIcon Name="heart" Class="text-red-500" />
<HeroIcon Name="star" Variant="HeroIconVariant.Solid" Class="text-yellow-400" />
<FeatherIcon Name="check" Class="text-green-600" />

@* Size (alternative to Size parameter) *@
<LucideIcon Name="home" Class="w-4 h-4" />
<HeroIcon Name="home" Class="w-6 h-6" />
<FeatherIcon Name="home" Class="w-8 h-8" />

@* Rotation *@
<LucideIcon Name="arrow-right" Class="rotate-45" />

@* Animation *@
<LucideIcon Name="loader-2" Class="animate-spin" />
<FeatherIcon Name="bell" Class="animate-bounce" />

@* Hover effects *@
<LucideIcon Name="heart" Class="hover:text-red-500 transition-colors" />
```

### Using CSS Variables (Theming)

```razor
<LucideIcon Name="sun" Color="var(--primary)" />
<HeroIcon Name="moon" Color="var(--foreground)" />
<FeatherIcon Name="star" Color="var(--accent)" />
```

## Accessibility

### Adding ARIA Labels

For icons that convey meaning (not just decoration), add `AriaLabel`:

```razor
@* Icon-only button - needs aria-label *@
<Button Size="icon">
    <LucideIcon Name="x" Size="20" AriaLabel="Close" />
</Button>

@* Icon with text - no aria-label needed *@
<Button>
    <LucideIcon Name="save" Size="16" />
    Save
</Button>
```

### Decorative vs. Meaningful Icons

```razor
@* Decorative (icon + text) - no aria-label needed *@
<Button>
    <HeroIcon Name="arrow-down-tray" Variant="HeroIconVariant.Outline" Size="16" />
    Download
</Button>

@* Meaningful (icon only) - aria-label required *@
<Button Size="icon">
    <HeroIcon Name="arrow-down-tray" Size="20" AriaLabel="Download file" />
</Button>
```

## Icon Name Format

All icon libraries use **kebab-case** for icon names:

```razor
@* Correct ✓ *@
<LucideIcon Name="chevron-down" />
<HeroIcon Name="user-plus" />
<FeatherIcon Name="alert-triangle" />

@* Wrong ✗ *@
<LucideIcon Name="ChevronDown" />  @* PascalCase - won't work *@
<HeroIcon Name="userPlus" />       @* camelCase - won't work *@
<FeatherIcon Name="alertTriangle" /> @* camelCase - won't work *@
```

## Troubleshooting

### Icon doesn't appear

**Cause:** Icon name is incorrect or not in kebab-case

**Solution:**
- Check icon name on official website (lucide.dev, heroicons.com, or feathericons.com)
- Ensure name is kebab-case: `"chevron-down"` not `"ChevronDown"`
- Verify the correct icon package is installed
- Verify namespace is imported in `_Imports.razor`

### Wrong variant shows (Heroicons only)

**Cause:** Not specifying variant or using wrong variant name

**Solution:**
```razor
@* Specify variant explicitly *@
<HeroIcon Name="camera" Variant="HeroIconVariant.Solid" />
<HeroIcon Name="camera" Variant="HeroIconVariant.Mini" />
<HeroIcon Name="camera" Variant="HeroIconVariant.Micro" />
```

### Icon color doesn't change

**Cause:** Using both `Color` parameter and CSS color classes

**Solution:**
```razor
@* Choose ONE approach *@

@* Option 1: Use Color parameter *@
<LucideIcon Name="heart" Color="red" />

@* Option 2: Use CSS class (preferred for theming) *@
<LucideIcon Name="heart" Class="text-red-500" />
```
