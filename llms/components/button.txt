# Button Component

Interactive button with multiple visual variants, sizes, and states following the shadcn/ui design system.

---

## Overview

The Button component provides a customizable, accessible button that supports multiple visual styles, sizes, and states. It follows WCAG 2.1 AA standards for accessibility and integrates seamlessly with Blazor's event system.

**Features:**
- 6 visual variants (Default, Destructive, Outline, Secondary, Ghost, Link)
- 6 size options (Small, Default, Large, Icon, IconSmall, IconLarge)
- Disabled state support
- Icon support with configurable position
- Full keyboard navigation (Tab, Enter, Space)
- ARIA attributes for screen readers
- Dark mode compatible via CSS variables

## API Reference

### `<Button>`

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `Variant` | `ButtonVariant` | `Default` | Visual style (Default, Destructive, Outline, Secondary, Ghost, Link) |
| `Size` | `ButtonSize` | `Default` | Size (Small, Default, Large, Icon, IconSmall, IconLarge) |
| `Type` | `ButtonType` | `Button` | HTML type (Button, Submit, Reset) |
| `Disabled` | `bool` | `false` | Whether button is disabled |
| `Class` | `string?` | `null` | Additional CSS classes |
| `OnClick` | `EventCallback<MouseEventArgs>` | - | Click event handler |
| `ChildContent` | `RenderFragment?` | `null` | Button content (text, icons, markup) |
| `AriaLabel` | `string?` | `null` | ARIA label for accessibility (required for icon-only buttons) |
| `Icon` | `RenderFragment?` | `null` | Icon to display in button |
| `IconPosition` | `IconPosition` | `Start` | Icon position (Start or End) |

### Enums

**ButtonVariant:**
- `Default` - Primary action button with primary color
- `Destructive` - Dangerous/destructive actions (red)
- `Outline` - Secondary action with border
- `Secondary` - Less prominent actions
- `Ghost` - Minimal styling, transparent background
- `Link` - Link-styled button with underline on hover

**ButtonSize:**
- `Small` - Compact button (h-9, text-xs)
- `Default` - Standard button (h-10)
- `Large` - Larger button (h-11, px-8)
- `Icon` - Square icon button (h-10 w-10)
- `IconSmall` - Small square icon button (h-9 w-9)
- `IconLarge` - Large square icon button (h-11 w-11)

**ButtonType:**
- `Button` - Standard button (default, prevents form submission)
- `Submit` - Form submit button
- `Reset` - Form reset button

### Loading State

```razor
<Button Disabled="@isLoading">
    @if (isLoading)
    {
        <LucideIcon Name="loader-2" Size="16" Class="animate-spin" />
    }
    else
    {
        <LucideIcon Name="save" Size="16" />
    }
    @(isLoading ? "Saving..." : "Save")
</Button>

@code {
    private bool isLoading = false;

    private async Task HandleSave()
    {
        isLoading = true;
        // Perform save operation
        await Task.Delay(2000);
        isLoading = false;
    }
}
```

### Form Submit Button

```razor

    <Field>
        <FieldLabel>Email</FieldLabel>
        <FieldContent>
            <Input @bind-Value="model.Email" Type="email" />
        </FieldContent>
    </Field>

    @* Submit button - uses Type="Submit" *@
    <Button Type="ButtonType.Submit">
        Submit Form
    </Button>
</EditForm>

@code {
    private FormModel model = new();

    private async Task HandleSubmit()
    {
        // Handle form submission
    }

    public class FormModel
    {
        [Required]
        [EmailAddress]
        public string Email { get; set; } = "";
    }
}
```

## Accessibility

### Keyboard Navigation
- **Tab**: Focus button
- **Enter/Space**: Activate button
- **Tab** (when focused): Move to next focusable element

### ARIA Attributes
- `aria-label`: Set via `AriaLabel` parameter (required for icon-only buttons)
- `aria-disabled`: Automatically set when `Disabled="true"`
- `tabindex`: Automatically set to `-1` when disabled

### Screen Reader Support

```razor
@* Text button - accessible by default *@
<Button>Save Changes</Button>

@* Icon-only button - requires aria-label *@
<Button Size="ButtonSize.Icon" AriaLabel="Close dialog">
    <LucideIcon Name="x" Size="20" />
</Button>

@* Button with icon and text - aria-label optional *@
<Button>
    <LucideIcon Name="download" Size="16" />
    Download
</Button>
```

### Focus Management

```razor
@* Visible focus ring (built-in) *@
<Button>Click me</Button>

@* Custom focus styling *@
<Button Class="focus-visible:ring-4">
    Enhanced Focus
</Button>
```

## Common Patterns

### Confirm/Cancel Actions

```razor
<div class="flex space-x-2">
    <Button Variant="ButtonVariant.Outline" OnClick="Cancel">
        Cancel
    </Button>
    <Button OnClick="Confirm">
        Confirm
    </Button>
</div>
```

### Async Actions with Loading

```razor
<Button
    Disabled="@isProcessing"
    OnClick="ProcessAction">
    @if (isProcessing)
    {
        <LucideIcon Name="loader-2" Size="16" Class="animate-spin" />
        <span>Processing...</span>
    }
    else
    {
        <LucideIcon Name="check" Size="16" />
        <span>Process</span>
    }
</Button>

@code {
    private bool isProcessing = false;

    private async Task ProcessAction()
    {
        isProcessing = true;
        try
        {
            // Perform async operation
            await Task.Delay(2000);
        }
        finally
        {
            isProcessing = false;
        }
    }
}
```

### Navigation Button

```razor
@inject NavigationManager Navigation

<Button OnClick="@(() => Navigation.NavigateTo("/dashboard"))">
    <LucideIcon Name="home" Size="16" />
    Go to Dashboard
</Button>
```

## Troubleshooting

### Issue: Button doesn't submit form

**Solution:** Use `Type="ButtonType.Submit"` for form submission buttons. Default type is `Button` to prevent accidental submissions.

```razor
<EditForm Model="@model" OnValidSubmit="@HandleSubmit">
    <Button Type="ButtonType.Submit">Submit</Button>
</EditForm>
```

### Issue: OnClick not firing when disabled

**Solution:** This is expected behavior. The `OnClick` callback is not invoked when `Disabled="true"`. Remove the disabled state to enable clicking.

### Issue: Icon button has no visible label

**Solution:** Use `AriaLabel` for accessibility:

```razor
<Button Size="ButtonSize.Icon" AriaLabel="Add item">
    <LucideIcon Name="plus" Size="20" />
</Button>
```

### Issue: Custom classes not applying

**Solution:** Make sure classes are added via the `Class` parameter, not as attributes:

```razor
@* Correct *@
<Button Class="w-full">Button</Button>

@* Incorrect *@
<Button class="w-full">Button</Button>
```
